## 📚 How to Study Each Topic

### 1. Read the Theory (15-20 min)
- Understand concepts before coding
- Take notes on key points
- Draw diagrams if helpful

### 2. Type the Code (30-45 min)
- **Don't copy-paste!** Type every example
- Modify examples to test understanding
- Break things intentionally to learn

### 3. Build Projects (60-90 min)
- Use concepts in real applications
- Start simple, add features gradually
- Debug errors yourself first

### 4. Review & Reflect (15 min)
- What did you learn?
- What's still unclear?
- How would you explain it to someone?

## 🛠️ Setup Your Environment

### Essential Tools

```bash
# 1. Install Flutter
# Download from: https://flutter.dev/docs/get-started/install

# 2. Verify installation
flutter doctor

# 3. Install VS Code extensions
# - Flutter
# - Dart
# - Error Lens
# - Bracket Pair Colorizer

# 4. Create project structure
mkdir flutter-learning
cd flutter-learning

# 5. Create first project
flutter create counter_app
cd counter_app
flutter run
```

### Recommended VS Code Settings

```json
{
  "editor.formatOnSave": true,
  "editor.codeActionsOnSave": {
    "source.fixAll": true
  },
  "dart.previewFlutterUiGuides": true,
  "dart.debugExternalPackageLibraries": true,
  "dart.debugSdkLibraries": false
}
```

## 💡 Learning Tips

### Do's ✅
- **Code daily** - Even 30 minutes
- **Build projects** - Apply concepts immediately  
- **Make mistakes** - Best way to learn
- **Ask questions** - Use Stack Overflow, Discord
- **Read others' code** - GitHub, pub.dev
- **Debug yourself** - Read error messages carefully
- **Take breaks** - Pomodoro technique (25 min work, 5 min break)

### Don'ts ❌
- **Don't just watch tutorials** - Code along
- **Don't copy-paste** - Type everything
- **Don't skip basics** - They're foundation
- **Don't memorize** - Understand concepts
- **Don't rush** - Quality over speed
- **Don't get stuck too long** - Ask for help after 30 min

## 🎤 Interview Preparation

### 2 Weeks Before Interview:
- Review interview-questions.md thoroughly
- Practice coding challenges on screen
- Build a project from scratch in 2-4 hours
- Explain your code to someone

### 1 Week Before:
- Mock interviews with friends
- Review your projects

### Day Before:
- Light review of quick-reference.md
- Setup your coding environment
- Get good sleep!

### During Interview:
- Think out loud
- Ask clarifying questions
- Start simple, then optimize
- Test your code
